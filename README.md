# CodeReadme  

A command‑line tool that scans a source repository and generates a natural‑language README.md using LLM.

## Overview  
CodeReadme walks the current directory, reads all source files (respecting a `.codereadmeignore` file), builds a prompt describing the project, sends it to LLM, and writes the resulting documentation to `README.md`. It can create a new README or update an existing one.

## Features  
- Scans a Go module or any directory tree.  
- Ignores files listed in `.codereadmeignore`.  
- Generates a README via the LLM.  
- Writes or updates `README.md` in the project root.  
- Reads the API key from the `API_KEY` environment variable (or a `.env` file).  

## Tech Stack  
- **Language:** Go 1.26  
- **Dependencies:**  
  - `github.com/joho/godotenv` – loads environment variables from a `.env` file.  
- **Internal packages:**  
  - `scanner` – file discovery and ignore‑pattern handling.  
  - `parser` – builds the prompt from the scanned files.  
  - `writer` – creates or updates `README.md`.  
  - `llm` – HTTP client that calls the LLM.  

## Installation  
1. Ensure Go 1.26 or newer is installed.  
2. Clone the repository.  
3. (Optional) Create a `.env` file in the project root with:  

```
API_KEY=<your‑key>
AI_MODEL=<model_name>
API_ENDPOINT=<url_endpoint>
```

   The key value itself is **not** documented; only the variable name is required.  

4. Build or run directly:  

   ```bash
   go run ./...
   ```

   The binary will be placed in the current directory or can be installed with `go install`.

## Usage  
Run the tool with the `generate` command:

```bash
codereadme generate
```

The tool will read the codebase, contact LLM, and write (or replace) `README.md` in the project root. If `README.md` already exists, a backup named `README.old.md` is created before overwriting.

## Notes  
- The tool respects the `.codereadmeignore` file for file inclusion/exclusion, mirroring the behavior of `.gitignore`.  
- No internal implementation details are exposed to the end user; the README generation logic is encapsulated within the `generator` package.  
- Errors from the LLM (e.g., authentication failures, rate limits) are propagated as human‑readable messages.
